# Contributing to RoboSats Gateway

Thank you for your interest in contributing to RoboSats Gateway!

## How to Contribute

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature`
3. **Make your changes**
4. **Test the installation process**: `./install-robosats-gateway.sh`
5. **Commit your changes**: `git commit -am 'Add some feature'`
6. **Push to the branch**: `git push origin feature/your-feature`
7. **Submit a pull request**

## Development Guidelines

### Testing
- Always test the complete installation process
- Verify management scripts work correctly
- Test on a clean system when possible

### Code Style
- Use clear, descriptive commit messages
- Add comments for complex logic
- Follow existing code formatting

### Documentation
- Update README.md if you change functionality
- Update INSTALL.md for installation-related changes
- Include examples in your documentation

## Areas for Contribution

- **Platform Support**: Add support for other Linux distributions
- **Security Enhancements**: Improve security configurations
- **SSL/HTTPS**: Enhance certificate management
- **Monitoring**: Add health checks and monitoring tools
- **Configuration**: More customization options
- **Documentation**: Improve guides and troubleshooting

## Reporting Issues

When reporting issues, please include:
- Your operating system and version
- Docker and Docker Compose versions
- Complete error messages
- Steps to reproduce the issue

## Code of Conduct

This project follows the same code of conduct as the RoboSats project. Be respectful and constructive in all interactions.
